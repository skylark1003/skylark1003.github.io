189. Rotate Array

Posted on 2022-08-20 In leetcode

一、描述

Given an array, rotate the array to the right by k steps, where k is non-negative.

二、例子

Example 1:

Input: nums = [1,2,3,4,5,6,7], k = 3
Output: [5,6,7,1,2,3,4]
Explanation:
rotate 1 steps to the right: [7,1,2,3,4,5,6]
rotate 2 steps to the right: [6,7,1,2,3,4,5]
rotate 3 steps to the right: [5,6,7,1,2,3,4]

Example 2:

Input: nums = [-1,-100,3,99], k = 2
Output: [3,99,-1,-100]
Explanation:
rotate 1 steps to the right: [99,-1,-100,3]
rotate 2 steps to the right: [3,99,-1,-100]

Constraints:

  • 1 <= nums.length <= 105
  • -231 <= nums[i] <= 231 - 1
  • 0 <= k <= 105

Follow up:

  • Try to come up with as many solutions as you can. There are at least three different ways to solve this problem.
  • Could you do it in-place with O(1) extra space?

172. Factorial Trailing Zeroes

Posted on 2022-08-19 In leetcode

一、描述

Given an integer n, return the number of trailing zeroes in n!.

Note that n! = n * (n - 1) * (n - 2) * ... * 3 * 2 * 1.

二、例子

Example 1:

Input: n = 3
Output: 0
Explanation: 3! = 6, no trailing zero.

Example 2:

Input: n = 5
Output: 1
Explanation: 5! = 120, one trailing zero.

Example 3:

Input: n = 0
Output: 0

Constraints:

  • 0 <= n <= 104

三、代码

算法：

一个数 n 的阶乘末尾有多少个 0 取决于从 1 到 n 的各个数的因子中 2 和 5 的个数，而 2 的个数是远远多余 5 的个数的, 因此求出 5 的个数即可。

例：100
100/5 = 20, 20/5 = 4, 4/5 = 0
则 1 到 100 中因子 5 的个数为 (20 + 4 + 0) = 24 个

思路：

  • 首先加 n/5
  • 我们忘记了 5X5, 2X5X5…， 所以我们加上 n/25
    （不把每个看作两个5是因为在 n/5 中已经加过了）
  • 我们忘记了 5X5X5, 2X5X5X5…， 所以我们加上 n/125
  • 所以 count = n/5 + n/25 + n/125 + … + 0

166. Fraction to Recurring Decimal

Posted on 2022-08-18 In leetcode

一、描述

Given two integers representing the numerator and denominator of a fraction, return the fraction in string format.

If the fractional part is repeating, enclose the repeating part in parentheses.

If multiple answers are possible, return any of them.

It is guaranteed that the length of the answer string is less than 104 for all the given inputs.

二、例子

Example 1:

Input: numerator = 1, denominator = 2
Output: “0.5”

Example 2:

Input: numerator = 2, denominator = 1
Output: “2”

Example 3:

Input: numerator = 4, denominator = 333
Output: “0.(012)”

Constraints:

  • -231 <= numerator, denominator <= 231 - 1
  • denominator != 0

三、代码

思路：

  1. 如果答案是整数，则直接计算返回。
  2. 如果答案是小数
    1. 先获取整数部分
    2. 小数部分计算方法为 余数 / 除数 的值添加到小数部分，之后 新余数 = 余数 % 除数 * 10，如此循环。
    3. 直至
      • 余数为0（非循环小数）
      • 余数可以在数组中找到（循环体出现）
  3. 注意：
    1. 由于负数在后续运算会添加负号，所以提前取绝对值进行运算。
    2. INT_MIN在取绝对值时会溢出，所以用long进行存储。

例：1 / 8
整数部分：1 / 8 = 0
小数部分：
余数：1 % 8 * 10 = 10
小数部分：10 / 8 = 1
余数：10 % 8 * 10 = 20
小数部分：20 / 8 = 2
余数：20 % 8 * 10 = 40
小数部分：40 / 8 = 5
余数：40 % 8 = 0（循环结束）
∴ 1 / 8 = 0.125


例：1 / 6
整数部分：1 / 6 = 0
小数部分：
余数：1 % 6 * 10 = 10
数组：10
小数部分：10 / 6 = 1
余数：10 % 6 * 10 = 40
数组：10, 40
小数部分：40 % 6 * 10 = 40
40可以在数组中找到，说明进入循环，且从下标为1处开始循环。
∴ 1 / 6 = 0.1(6)

162. Find Peak Element

Posted on 2022-08-18 In leetcode

一、描述

A peak element is an element that is strictly greater than its neighbors.

Given a 0-indexed integer array nums, find a peak element, and return its index. If the array contains multiple peaks, return the index to any of the peaks.

You may imagine that nums[-1] = nums[n] = -∞. In other words, an element is always considered to be strictly greater than a neighbor that is outside the array.

You must write an algorithm that runs in O(log n) time.

二、例子

Example 1:

Input: nums = [1,2,3,1]
Output: 2
Explanation: 3 is a peak element and your function should return the index number 2.

Example 2:

Input: nums = [1,2,1,3,5,6,4]
Output: 5
Explanation: Your function can return either index number 1 where the peak element is 2, or index number 5 where the peak element is 6.

Constraints:

  • 1 <= nums.length <= 1000
  • -231 <= nums[i] <= 231 - 1
  • nums[i] != nums[i + 1] for all valid i.
