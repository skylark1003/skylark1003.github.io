207. Course Schedule

Posted on 2022-08-22 Edited on 2022-08-23 In leetcode

一、描述

There are a total of numCourses courses you have to take, labeled from 0 to numCourses - 1. You are given an array prerequisites where prerequisites[i] = [ai, bi] indicates that you must take course bi first if you want to take course ai.

  • For example, the pair [0, 1], indicates that to take course 0 you have to first take course 1.

Return true if you can finish all courses. Otherwise, return false.

二、例子

Example 1:

Input: numCourses = 2, prerequisites = [[1,0]]
Output: true
Explanation: There are a total of 2 courses to take.
To take course 1 you should have finished course 0. So it is possible.

Example 2:

Input: numCourses = 2, prerequisites = [[1,0],[0,1]]
Output: false
Explanation: There are a total of 2 courses to take.
To take course 1 you should have finished course 0, and to take course 0 you should also have finished course 1. So it is impossible.

Constraints:

  • 1 <= numCourses <= 2000
  • 0 <= prerequisites.length <= 5000
  • prerequisites[i].length == 2
  • 0 <= ai, bi < numCourses
  • All the pairs prerequisites[i] are unique.

三、代码

解法超时：
不应该用map存储，每次搜索都要花费时间。
应该利用数组的随机存储特性。

解法来自ciguvi：
填色算法

  • vis[id]=0表示尚未访问的节点
  • vis[id]=1表示当前访问的节点且当前其子节点正在被访问
  • vis[id]=2表示其所有子节点均被访问

Implement the MyStack class:

  • void push(int x) Pushes element x to the top of the stack.
  • int pop() Removes the element on the top of the stack and returns it.
  • int top() Returns the element on the top of the stack.
  • boolean empty() Returns true if the stack is empty, false otherwise.

Notes:

  • You must use only standard operations of a queue, which means that only push to back, peek/pop from front, size and is empty operations are valid.
  • Depending on your language, the queue may not be supported natively. You may simulate a queue using a list or deque (double-ended queue) as long as you use only a queue’s standard operations.

二、例子

Example 1:

Input

[“MyStack”, “push”, “push”, “top”, “pop”, “empty”]
[[], [1], [2], [], [], []]
Output
[null, null, null, 2, 2, false]

Explanation
MyStack myStack = new MyStack();
myStack.push(1);
myStack.push(2);
myStack.top(); // return 2
myStack.pop(); // return 2
myStack.empty(); // return False

Constraints:

  • 1 <= x <= 9
  • At most 100 calls will be made to push, pop, top, and empty.
  • All the calls to pop and top are valid.

【笔记】计算素数个数

Posted on 2022-08-22 In 笔记

一、题目

给一个整数n，返回小于n的素数个数。

二、算法

1.普通算法

从一个isPrime函数开始。为了确定一个数字是否为素数，我们需要检查它是否不能被任何小于n的数字整除。
isPrime函数的运行时复杂度为O(n)，因此将总素数计算n为O(n2)。

2.普通算法优化

  • 因为我们知道这个数字不能被任何>n/2的数字整除，所以我们可以将总的迭代减少一半，只除以到n/2。

  • 更进一步，我们只需要考虑到√n的因数，因为，如果n可以被某个数p整除，那么由于n=p×q且p≤q，我们可以推导出p≤√n，所以可以只除以到 √n。

3.埃拉托色尼筛选法

维基百科：Sieve of Eratosthenes

  • 这是一个包含n个数字的表。
  • 从第一个数字2开始，2的所有倍数都不能是素数，所以我们把它们标记为非素数。
  • 下一个数字3，同样地，所有3的倍数，如3×2=6,3×3=9，… 一定不能是素数，所以我们也把它们标记。
  • 下一个数字4，它已经被划掉了。4不是质数，因为它可以被2整除，这意味着4的所有倍数也必须可以被2整除，并且已经被划掉了。所以我们可以立即跳过4，进入下一个数字。
  • 下一个数字5，所有5的倍数，如5×2=10,5×3=15,5×4=20,5×5=25，……可以被标记掉。
    • 这里有一个小小的优化，我们不需要从5×2=10开始。
    • 可以从5×5=25开始标记5的倍数，因为5×2=10已经被2的倍数标记，同样，5×3=15已经被3的倍数标记。因此，如果当前的数字是p，我们总是可以从p2开始标记p的倍数，然后以p2p，p2+p，p2+2p为增量。
  • 终止循环条件可以是p<√n，因为所有非素数≥√n必须已经被标记关闭。当循环终止时，表中所有未标记的数字都为素数。

埃拉托色尼筛选法使用了一个额外的**O(n)内存，它的运行时复杂度为O(nloglogn)**。

You are a professional robber planning to rob houses along a street. Each house has a certain amount of money stashed, the only constraint stopping you from robbing each of them is that adjacent houses have security systems connected and it will automatically contact the police if two adjacent houses were broken into on the same night.

Given an integer array nums representing the amount of money of each house, return the maximum amount of money you can rob tonight without alerting the police.

二、例子

Example 1:

Input: nums = [1,2,3,1]
Output: 4
Explanation: Rob house 1 (money = 1) and then rob house 3 (money = 3).
Total amount you can rob = 1 + 3 = 4.

Example 2:

Input: nums = [2,7,9,3,1]
Output: 12
Explanation: Rob house 1 (money = 2), rob house 3 (money = 9) and rob house 5 (money = 1).
Total amount you can rob = 2 + 9 + 1 = 12.

Constraints:

  • 1 <= nums.length <= 100
  • 0 <= nums[i] <= 400
